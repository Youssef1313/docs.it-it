# <a name="nullconstant-is-not-declared"></a>' \<nullconstant >' non è dichiarato
' \<nullconstant >' non è dichiarato. La costante Null non è più supportata. Usare System.DBNull.  
  
 Un'istruzione usa la parola chiave `Null` , che non è più supportata in Visual Basic.  
  
 **ID errore:** BC30822  
  
## <a name="to-correct-this-error"></a>Per correggere l'errore  
  
1. Usare <xref:System.DBNull> anziché `Null`. Nell'esempio che segue viene illustrato quanto descritto.  
  
    ```vb  
    Sub TestDBNull()  
        Dim t As DataTable  
        ' Assume the DataGrid is bound to a DataTable.  
        t = CType(DataGrid1.DataSource, DataTable)  
        Dim r As DataRow  
        r = t.Rows(datagrid1.CurrentCell.RowNumber)  
        r.BeginEdit  
        r(1) = System.DBNull.Value ' Assign DBNull to the record.  
        r.EndEdit  
        r.AcceptChanges  
        If r.IsNull(1) Then  
            MsgBox("")  
        End If  
    End Sub  
    ```  
  
2. Utilizzare la parola chiave [Nothing](../../visual-basic/language-reference/nothing.md) per le assegnazioni e i confronti quando si utilizzano le variabili oggetto. Nell'esempio che segue viene illustrato quanto descritto.  
  
    ```vb  
    Sub TestNothing()  
        Dim cls As Object  
        ' cls is Nothing if it has not been assigned using the New keyword.  
        If (cls Is Nothing) Then  
            MsgBox("cls is Nothing")  
        End If  
        cls = Nothing ' Assign Nothing to the class variable cls.  
    End Sub  
    ```
