## <a name="example"></a><span data-ttu-id="8d97d-108">Esempio</span><span class="sxs-lookup"><span data-stu-id="8d97d-108">Example</span></span>

<span data-ttu-id="8d97d-109">Il codice seguente genera l'errore CS1941 perché all'operatore `equals` viene richiesto di confrontare un valore `int` con un valore `string`.</span><span class="sxs-lookup"><span data-stu-id="8d97d-109">The following code generates CS1941 because the `equals` operator is being asked to compare an `int` to a `string`.</span></span>  

```csharp
// cs1941.cs  
using System.Collections;  
using System.Linq;  
class Test  
{  
    static int Main()  
    {  
        var nums = new[] { 1, 2, 3, 4, 5, 6 };  
        var words = new string[] { "lake", "mountain", "sky" };  
        IEnumerable e = from n in nums  
                        join w in words on n equals w // CS1941  
                        select w;  
        return 0;  
    }  
}  
```

<span data-ttu-id="8d97d-110">Il metodo in cui l'inferenza del tipo non riesce è il metodo in cui viene convertita la clausola di query in fase di compilazione.</span><span class="sxs-lookup"><span data-stu-id="8d97d-110">The method in which type inference fails is the method that the query clause is translated to at compile time.</span></span>
