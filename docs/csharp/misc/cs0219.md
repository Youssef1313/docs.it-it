# <a name="compiler-warning-level-3-cs0219"></a><span data-ttu-id="fc4db-102">Avviso del compilatore (livello 3) CS0219</span><span class="sxs-lookup"><span data-stu-id="fc4db-102">Compiler Warning (level 3) CS0219</span></span>
<span data-ttu-id="fc4db-103">La variabile 'variable' è assegnata, ma il suo valore non viene mai usato</span><span class="sxs-lookup"><span data-stu-id="fc4db-103">The variable 'variable' is assigned but its value is never used</span></span>

 <span data-ttu-id="fc4db-104">Il compilatore genera un messaggio di avviso di livello 3 quando una variabile viene dichiarata e assegnata ma non viene mai usata.</span><span class="sxs-lookup"><span data-stu-id="fc4db-104">The compiler issues a level-three warning, when you declare and assign a variable, but do not use it.</span></span>

 > [!NOTE]
 > <span data-ttu-id="fc4db-105">Il compilatore genera questo avviso solo quando il valore della variabile è una costante in fase di compilazione.</span><span class="sxs-lookup"><span data-stu-id="fc4db-105">The compiler generates this warning only when the variable value is a compile-time constant.</span></span> <span data-ttu-id="fc4db-106">L'assegnazione di un risultato di un'espressione o di un metodo non costante a una variabile locale rende più semplice l'osservazione di tali espressioni nel debugger.</span><span class="sxs-lookup"><span data-stu-id="fc4db-106">Assigning a non-constant expression or method result to a local variable makes it easier to observe those expressions in the debugger.</span></span> <span data-ttu-id="fc4db-107">Rende anche raggiungibile il risultato, impedendo Garbage Collection mentre tale variabile è raggiungibile.</span><span class="sxs-lookup"><span data-stu-id="fc4db-107">It also makes the result reachable, preventing garbage collection while that variable is reachable.</span></span>
 
 <span data-ttu-id="fc4db-108">Nell'esempio seguente vengono illustrati i casi in cui e quando non viene generato l'avviso:</span><span class="sxs-lookup"><span data-stu-id="fc4db-108">The following sample shows the cases when and when not the warning is generated:</span></span>

```csharp
// CS0219.cs
// compile with: /W:3
public class MyClass
{
   public static void Main()
   {
      var interpolated = "Interpolated";
      var a = 0; // CS0219
      int b = GetZero(); // Doesn't generate a warning.
      var c = "Regular string"; // CS0219
      var d = $"Constant interpolated string"; // Doesn't generate a warning.
      var e = $"{interpolated} string"; // Doesn't generate a warning.
   }
   
   private static int GetZero()
   {
       return 0;
   }
}  
```
