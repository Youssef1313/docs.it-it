# <a name="compiler-warning-level-1-cs1570"></a>Avviso del compilatore (livello 1) CS1570
Il commento XML in 'construct' presenta un formato XML errato - 'reason'  
  
 Quando si usa [/doc](../language-reference/compiler-options/doc-compiler-option.md), i commenti nel codice sorgente devono avere il formato XML. Gli errori relativi al markup XML generano l'avviso CS1570. Ad esempio:  
  
- Se si passa una stringa a un **cref**, ad esempio in un' [ \<eccezione >](../programming-guide/xmldoc/exception.md) tag, la stringa deve essere racchiusa tra virgolette doppie.  
  
- Se si usa un tag, ad esempio [ \<seealso >](../programming-guide/xmldoc/seealso.md), che non dispone di un tag di chiusura, è necessario specificare una barra prima della parentesi angolare di chiusura.  
  
- Per usare i simboli maggiore di o minore di nel testo della descrizione, è necessario indicarli con **&gt;** o **&lt;** .  
  
- L'attributo del percorso o del file [ \<](../programming-guide/xmldoc/include.md) in un tag di inclusione > è mancante o è stato formattato in modo errato.  
  
 L'esempio seguente genera l'errore CS1570:  
  
```csharp  
// CS1570.cs  
// compile with: /W:1  
namespace ns  
{  
   // the following line generates CS1570  
   /// <summary> returns true if < 5 </summary>  
   // try this instead  
   // /// <summary> returns true if <5 </summary>  
  
   public class MyClass  
   {  
      public static void Main ()  
      {  
      }  
   }  
}  
```
