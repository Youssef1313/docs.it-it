# <a name="compiler-error-cs5001"></a>Errore del compilatore CS5001

Il programma non contiene un metodo ' Main ' statico appropriato per un punto di ingresso

Questo errore si verifica quando nel codice che produce un file eseguibile non viene trovato alcun metodo statico `Main` con una firma corretta. Si verifica anche se la funzione del punto di ingresso, `Main`, viene definita con il caso errato, ad esempio in minuscolo `main`. Per informazioni sulle regole che si applicano al metodo `Main`, vedere [Main () e argomenti della riga di comando](../programming-guide/main-and-command-args/index.md).

Se il metodo `Main` ha un modificatore `async`, verificare che la [versione C# della lingua selezionata](../language-reference/configure-language-version.md) sia 7,1 o successiva.

Il metodo `Main` Ã¨ necessario solo quando si compila un file eseguibile, ovvero quando si specifica l'opzione del compilatore When [-target: exe](/dotnet/csharp/language-reference/compiler-options/target-exe-compiler-option) o [-target: winexe](/dotnet/csharp/language-reference/compiler-options/target-winexe-compiler-option) . Per impostazione predefinita, i seguenti tipi di progetto di Visual Studio specificano una di queste opzioni:

- Applicazione console
- Applicazione ASP.NET Core
- applicazione WPF
- Windows Form (applicazione)

## <a name="example"></a>Esempio

L'esempio seguente genera l'errore CS5001:
  
```csharp
// CS5001.cs
// CS5001 expected when compiled with -target:exe or -target:winexe
public class Program
{
   // Uncomment the following line to resolve.
   // static void Main() {}
}
```
