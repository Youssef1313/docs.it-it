# <a name="how-to-create-unsigned-friend-assemblies-c"></a><span data-ttu-id="cca66-102">Procedura: Creare assembly Friend non firmati (C#)</span><span class="sxs-lookup"><span data-stu-id="cca66-102">How to: Create Unsigned Friend Assemblies (C#)</span></span>
<span data-ttu-id="cca66-103">In questo esempio viene illustrato come usare assembly Friend e assembly non firmati.</span><span class="sxs-lookup"><span data-stu-id="cca66-103">This example shows how to use friend assemblies with assemblies that are unsigned.</span></span>  
  
### <a name="to-create-an-assembly-and-a-friend-assembly"></a><span data-ttu-id="cca66-104">Per creare un assembly e un assembly Friend</span><span class="sxs-lookup"><span data-stu-id="cca66-104">To create an assembly and a friend assembly</span></span>  
  
1. <span data-ttu-id="cca66-105">Aprire un prompt dei comandi.</span><span class="sxs-lookup"><span data-stu-id="cca66-105">Open a command prompt.</span></span>  
  
2. <span data-ttu-id="cca66-106">Creare un file C# denominato `friend_unsigned_A.` contenente il codice seguente.</span><span class="sxs-lookup"><span data-stu-id="cca66-106">Create a C# file named `friend_unsigned_A.` that contains the following code.</span></span> <span data-ttu-id="cca66-107">Il codice usa l'attributo <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> per dichiarare friend_unsigned_B come assembly Friend.</span><span class="sxs-lookup"><span data-stu-id="cca66-107">The code uses the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute to declare friend_unsigned_B as a friend assembly.</span></span>  
  
    ```csharp  
    // friend_unsigned_A.cs  
    // Compile with:   
    // csc /target:library friend_unsigned_A.cs  
    using System.Runtime.CompilerServices;  
    using System;  
  
    [assembly: InternalsVisibleTo("friend_unsigned_B")]  
  
    // Type is internal by default.  
    class Class1  
    {  
        public void Test()  
        {  
            Console.WriteLine("Class1.Test");  
        }  
    }  
  
    // Public type with internal member.  
    public class Class2  
    {  
        internal void Test()  
        {  
            Console.WriteLine("Class2.Test");  
        }  
    }  
    ```  
  
3. <span data-ttu-id="cca66-108">Compilare e firmare friend_unsigned_A usando il comando seguente.</span><span class="sxs-lookup"><span data-stu-id="cca66-108">Compile and sign friend_unsigned_A by using the following command.</span></span>  
  
    ```csharp  
    csc /target:library friend_unsigned_A.cs  
    ```  
  
4. <span data-ttu-id="cca66-109">Creare un file C# denominato `friend_unsigned_B` contenente il codice seguente.</span><span class="sxs-lookup"><span data-stu-id="cca66-109">Create a C# file named `friend_unsigned_B` that contains the following code.</span></span> <span data-ttu-id="cca66-110">Poiché friend_unsigned_A specifica che friend_unsigned_B è un assembly Friend, il codice in friend_unsigned_B può accedere ai tipi e ai membri `internal` da friend_unsigned_A.</span><span class="sxs-lookup"><span data-stu-id="cca66-110">Because friend_unsigned_A specifies friend_unsigned_B as a friend assembly, the code in friend_unsigned_B can access `internal` types and members from friend_unsigned_A.</span></span>  
  
    ```csharp  
    // friend_unsigned_B.cs  
    // Compile with:   
    // csc /r:friend_unsigned_A.dll /out:friend_unsigned_B.exe friend_unsigned_B.cs  
    public class Program  
    {  
        static void Main()  
        {  
            // Access an internal type.  
            Class1 inst1 = new Class1();  
            inst1.Test();  
  
            Class2 inst2 = new Class2();  
            // Access an internal member of a public type.  
            inst2.Test();  
  
            System.Console.ReadLine();  
        }  
    }  
    ```  
  
5. <span data-ttu-id="cca66-111">Compilare friend_unsigned_B usando il comando seguente.</span><span class="sxs-lookup"><span data-stu-id="cca66-111">Compile friend_unsigned_B by using the following command.</span></span>  
  
    ```csharp  
    csc /r:friend_unsigned_A.dll /out:friend_unsigned_B.exe friend_unsigned_B.cs  
    ```  
  
     <span data-ttu-id="cca66-112">Il nome dell'assembly generato dal compilatore deve corrispondere al nome dell'assembly Friend passato all'attributo <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute>.</span><span class="sxs-lookup"><span data-stu-id="cca66-112">The name of the assembly that is generated by the compiler must match the friend assembly name that is passed to the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute.</span></span> <span data-ttu-id="cca66-113">È necessario specificare in modo esplicito il nome dell'assembly di output (con estensione exe o dll) usando il compilatore `/out`.</span><span class="sxs-lookup"><span data-stu-id="cca66-113">You must explicitly specify the name of the output assembly (.exe or .dll) by using the `/out` compiler option.</span></span> <span data-ttu-id="cca66-114">Per altre informazioni, vedere [/out (Opzioni del compilatore C#)](../../../language-reference/compiler-options/out-compiler-option.md).</span><span class="sxs-lookup"><span data-stu-id="cca66-114">For more information, see [/out (C# Compiler Options)](../../../language-reference/compiler-options/out-compiler-option.md).</span></span>  
  
6. <span data-ttu-id="cca66-115">Eseguire il file friend_unsigned_B.exe.</span><span class="sxs-lookup"><span data-stu-id="cca66-115">Run the friend_unsigned_B.exe file.</span></span>  
  
     <span data-ttu-id="cca66-116">Il programma stampa due stringhe: "Class1.Test" e "Class2.Test".</span><span class="sxs-lookup"><span data-stu-id="cca66-116">The program prints two strings: "Class1.Test" and "Class2.Test".</span></span>  
  
## <a name="net-framework-security"></a><span data-ttu-id="cca66-117">Sicurezza di .NET Framework</span><span class="sxs-lookup"><span data-stu-id="cca66-117">.NET Framework Security</span></span>  
 <span data-ttu-id="cca66-118">Ci sono alcune analogie tra l'attributo <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> e la classe <xref:System.Security.Permissions.StrongNameIdentityPermission>.</span><span class="sxs-lookup"><span data-stu-id="cca66-118">There are similarities between the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute and the <xref:System.Security.Permissions.StrongNameIdentityPermission> class.</span></span> <span data-ttu-id="cca66-119">La differenza principale è che <xref:System.Security.Permissions.StrongNameIdentityPermission> può chiedere le autorizzazioni di sicurezza per l'esecuzione di una particolare sezione di codice, mentre l'attributo <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> controlla la visibilità dei membri e dei tipi `internal`.</span><span class="sxs-lookup"><span data-stu-id="cca66-119">The main difference is that <xref:System.Security.Permissions.StrongNameIdentityPermission> can demand security permissions to run a particular section of code, whereas the <xref:System.Runtime.CompilerServices.InternalsVisibleToAttribute> attribute controls the visibility of `internal` types and members.</span></span>
