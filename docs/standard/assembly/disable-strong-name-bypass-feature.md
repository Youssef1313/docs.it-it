 <span data-ttu-id="0e2bd-109">La funzionalità che consente di ignorare il nome sicuro si applica a qualsiasi assembly ad attendibilità totale che non abbia la firma ritardata e che sia caricato in un oggetto <xref:System.AppDomain> ad attendibilità totale dalla directory specificata dalla proprietà <xref:System.AppDomainSetup.ApplicationBase%2A>.</span><span class="sxs-lookup"><span data-stu-id="0e2bd-109">The bypass feature applies to any full-trust assembly that is not delay-signed and that is loaded into any full-trust <xref:System.AppDomain> from the directory specified by its <xref:System.AppDomainSetup.ApplicationBase%2A> property.</span></span>

## <a name="disable-the-strong-name-bypass-feature-for-all-applications"></a><span data-ttu-id="0e2bd-118">Disabilitare la funzionalità di bypass con nome sicuro per tutte le applicazioni</span><span class="sxs-lookup"><span data-stu-id="0e2bd-118">Disable the strong-name bypass feature for all applications</span></span>  
  
- <span data-ttu-id="0e2bd-119">Nei computer a 32 bit creare nel Registro di sistema una voce DWORD con valore 0 denominata `AllowStrongNameBypass` nella chiave HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\\.NETFramework.</span><span class="sxs-lookup"><span data-stu-id="0e2bd-119">On 32-bit computers, in the system registry, create a DWORD entry with a value of 0 named `AllowStrongNameBypass` under the HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\\.NETFramework key.</span></span>  
  
- <span data-ttu-id="0e2bd-120">Nei computer a 64 bit creare nel Registro di sistema una voce DWORD con valore 0 denominata `AllowStrongNameBypass` nelle chiavi HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\\.NETFramework e HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\\.NETFramework.</span><span class="sxs-lookup"><span data-stu-id="0e2bd-120">On 64-bit computers, in the system registry, create a DWORD entry with a value of 0 named `AllowStrongNameBypass` under the HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\\.NETFramework and HKEY_LOCAL_MACHINE\SOFTWARE\Wow6432Node\Microsoft\\.NETFramework keys.</span></span>  
  
## <a name="disable-the-strong-name-bypass-feature-for-a-single-application"></a><span data-ttu-id="0e2bd-121">Disabilitare la funzionalità di bypass con nome sicuro per una singola applicazione</span><span class="sxs-lookup"><span data-stu-id="0e2bd-121">Disable the strong-name bypass feature for a single application</span></span>  
  
1. <span data-ttu-id="0e2bd-122">Aprire o creare il file di configurazione dell'applicazione.</span><span class="sxs-lookup"><span data-stu-id="0e2bd-122">Open or create the application configuration file.</span></span>  
  
    <span data-ttu-id="0e2bd-123">Per altre informazioni su questo file, vedere la sezione file di configurazione dell'applicazione in [configurare le app](../../framework/configure-apps/index.md).</span><span class="sxs-lookup"><span data-stu-id="0e2bd-123">For more information about this file, see the Application Configuration Files section in [Configure apps](../../framework/configure-apps/index.md).</span></span>

    ```xml  
    <configuration>  
      <runtime>  
        <bypassTrustedAppStrongNames enabled="false" />  
      </runtime>  
    </configuration>  
    ```  
  
 <span data-ttu-id="0e2bd-125">È possibile ripristinare la funzionalità bypass per l'applicazione rimuovendo l'impostazione del file di configurazione o impostando l'attributo su `true`.</span><span class="sxs-lookup"><span data-stu-id="0e2bd-125">You can restore the bypass feature for the application by removing the configuration file setting or by setting the attribute to `true`.</span></span>
