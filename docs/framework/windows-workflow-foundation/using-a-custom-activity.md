# <a name="using-a-custom-activity"></a><span data-ttu-id="7b87e-102">Utilizzo di un'attività personalizzata</span><span class="sxs-lookup"><span data-stu-id="7b87e-102">Using a custom activity</span></span>
<span data-ttu-id="7b87e-103">Le attività che derivano da <xref:System.Activities.Activity> o le relative sottoclassi possono essere composte in flussi di lavoro più grandi o create direttamente nel codice.</span><span class="sxs-lookup"><span data-stu-id="7b87e-103">Activities that derive from <xref:System.Activities.Activity> or its subclasses can be composed into larger workflows, or created directly in code.</span></span> <span data-ttu-id="7b87e-104">In questo argomento viene descritto come usare le attività personalizzate nei flussi di lavoro creati nel codice o nella finestra di progettazione.</span><span class="sxs-lookup"><span data-stu-id="7b87e-104">This topic describes how to use custom activities in workflows created either in code or in the designer.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="7b87e-105">È possibile utilizzare attività personalizzate nello stesso progetto in cui sono definite, purché sia l'attività personalizzata che l'attività che lo utilizza vengano compilate, ad esempio caricate da un tipo di creazione di istanze generato dal processo di compilazione, se l'attività di riferimento viene caricata. in modo dinamico (ad esempio, usando ActivityXAMLServices), l'assembly a cui si fa riferimento deve essere inserito in un progetto diverso o il codice XAML generato dalla finestra di progettazione deve essere modificato manualmente per abilitare questa operazione.</span><span class="sxs-lookup"><span data-stu-id="7b87e-105">Custom activities can be used in the same project in which they are defined, as long as both the custom activity and the activity that uses it are compiled (i.e. loaded by an instantiating type generated by the build process) If the referencing activity is loaded dynamically (e.g. using ActivityXAMLServices), then the referenced assembly should be placed in a different project, or the designer-generated XAML needs to be hand-edited to enable this.</span></span>  
  
#### <a name="using-a-custom-activity-to-a-workflow-project"></a><span data-ttu-id="7b87e-106">Uso di un'attività personalizzata per un progetto di flusso di lavoro</span><span class="sxs-lookup"><span data-stu-id="7b87e-106">Using a custom activity to a workflow project</span></span>  
  
1. <span data-ttu-id="7b87e-107">Aggiungere un riferimento dal progetto host al progetto della libreria di attività contenente l'attività personalizzata.</span><span class="sxs-lookup"><span data-stu-id="7b87e-107">Add a reference from the host project to the activity library project containing the custom activity.</span></span>  
  
2. <span data-ttu-id="7b87e-108">Compilare la soluzione.</span><span class="sxs-lookup"><span data-stu-id="7b87e-108">Build the solution.</span></span>  
  
3. <span data-ttu-id="7b87e-109">Per usare l'attività personalizzata nella finestra di progettazione, individuare l'attività personalizzata nella casella degli strumenti e trascinare l'attività nell'area di progettazione.</span><span class="sxs-lookup"><span data-stu-id="7b87e-109">To use the custom activity in the designer, locate the custom activity in the toolbox, and drag the activity onto the designer surface.</span></span>  
  
4. <span data-ttu-id="7b87e-110">Per usare l'attività personalizzata nel codice, aggiungere un'istruzione Using che fa riferimento al progetto di attività personalizzata e passare una nuova istanza dell'attività a <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span><span class="sxs-lookup"><span data-stu-id="7b87e-110">To use the custom activity in code, add a Using statement that refers to the custom activity project, and pass a new instance of the activity to <xref:System.Activities.WorkflowInvoker.Invoke%2A>.</span></span>
