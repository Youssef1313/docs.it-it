# <a name="deploying-services"></a>Distribuzione di servizi
In questo argomento viene descritto come distribuire un'applicazione Windows Communication Foundation (WCF) in un ambiente di Runtime.  
  
## <a name="choosing-the-hosting-environment-for-your-application"></a>Scelta dell'ambiente di hosting per l'applicazione  
 I servizi WCF sono progettati per essere eseguiti in qualsiasi processo di Windows che supporta il codice gestito. Per diventare attivo, un servizio deve essere ospitato all'interno di un ambiente di runtime che lo crea e ne controlla il contesto e la durata. Le opzioni di hosting vanno dall'esecuzione in una semplice applicazione console ad ambienti server come un servizio Windows, Internet Information Services (IIS) o all'interno di un processo di lavoro gestito da Windows Activation Service (WAS). Per esaminare le diverse opzioni di hosting per l'applicazione WCF, vedere [servizi di hosting](../hosting-services.md).
