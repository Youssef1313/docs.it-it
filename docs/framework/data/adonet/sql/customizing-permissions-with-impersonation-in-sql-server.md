# <a name="customizing-permissions-with-impersonation-in-sql-server"></a><span data-ttu-id="5ee9a-102">Personalizzazione delle autorizzazioni con rappresentazione in SQL Server</span><span class="sxs-lookup"><span data-stu-id="5ee9a-102">Customizing Permissions with Impersonation in SQL Server</span></span>
<span data-ttu-id="5ee9a-103">In molte applicazioni vengono usate le stored procedure per accedere ai dati, basandosi sul concatenamento delle proprietà per restringere l'accesso alle tabelle di base.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-103">Many applications use stored procedures to access data, relying on ownership chaining to restrict access to base tables.</span></span> <span data-ttu-id="5ee9a-104">È possibile concedere autorizzazioni EXECUTE sulle stored procedure, revocando o negando le autorizzazioni sulle tabelle di base.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-104">You can grant EXECUTE permissions on stored procedures, revoking or denying permissions on the base tables.</span></span> <span data-ttu-id="5ee9a-105">SQL Server non verifica le autorizzazioni del chiamante se il proprietario della stored procedure coincide con quello delle tabelle.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-105">SQL Server does not check the permissions of the caller if the stored procedure and tables have the same owner.</span></span> <span data-ttu-id="5ee9a-106">Il concatenamento delle proprietà non funziona se i proprietari degli oggetti sono diversi oppure se si usano istruzioni SQL dinamiche.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-106">However, ownership chaining doesn't work if objects have different owners or in the case of dynamic SQL.</span></span>  
  
 <span data-ttu-id="5ee9a-107">È possibile usare la clausola EXECUTE AS in una stored procedure quando il chiamante non dispone delle autorizzazioni sugli oggetti di database cui viene fatto riferimento.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-107">You can use the EXECUTE AS clause in a stored procedure when the caller doesn't have permissions on the referenced database objects.</span></span> <span data-ttu-id="5ee9a-108">Per effetto della clausola EXECUTE AS il contesto di esecuzione viene passato all'utente proxy.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-108">The effect of the EXECUTE AS clause is that the execution context is switched to the proxy user.</span></span> <span data-ttu-id="5ee9a-109">Tutto il codice, nonché qualsiasi chiamata a stored procedure o trigger annidati, viene eseguito nel contesto di sicurezza dell'utente proxy.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-109">All code, as well as any calls to nested stored procedures or triggers, executes under the security context of the proxy user.</span></span> <span data-ttu-id="5ee9a-110">Viene ripristinato il contesto di esecuzione del chiamante originale solo dopo l'esecuzione della stored procedure o quando viene eseguita un'istruzione REVERT.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-110">Execution context is reverted to the original caller only after execution of the procedure or when a REVERT statement is issued.</span></span>  
  
## <a name="context-switching-with-the-execute-as-statement"></a><span data-ttu-id="5ee9a-111">Cambio del contesto con l'istruzione EXECUTE AS</span><span class="sxs-lookup"><span data-stu-id="5ee9a-111">Context Switching with the EXECUTE AS Statement</span></span>  
 <span data-ttu-id="5ee9a-112">L'istruzione EXECUTE AS Transact-SQL consente di cambiare il contesto di esecuzione di un'istruzione mediante la rappresentazione di un altro account di accesso o utente del database.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-112">The Transact-SQL EXECUTE AS statement allows you to switch the execution context of a statement by impersonating another login or database user.</span></span> <span data-ttu-id="5ee9a-113">Si tratta di una tecnica utile per testare query e stored procedure usando le credenziali di un altro utente.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-113">This is a useful technique for testing queries and procedures as another user.</span></span>  
  
```  
EXECUTE AS LOGIN = 'loginName';  
EXECUTE AS USER = 'userName';  
```  
  
 <span data-ttu-id="5ee9a-114">È necessario disporre delle autorizzazioni IMPERSONATE per l'account di accesso o l'utente che si intende rappresentare.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-114">You must have IMPERSONATE permissions on the login or user you are impersonating.</span></span> <span data-ttu-id="5ee9a-115">Questa autorizzazione è implica per `sysadmin` su tutti i database e per i membri del ruolo `db_owner` sui database di cui sono proprietari.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-115">This permission is implied for `sysadmin` for all databases, and `db_owner` role members in databases that they own.</span></span>  
  
## <a name="granting-permissions-with-the-execute-as-clause"></a><span data-ttu-id="5ee9a-116">Concessione delle autorizzazioni con la clausola EXECUTE AS</span><span class="sxs-lookup"><span data-stu-id="5ee9a-116">Granting Permissions with the EXECUTE AS Clause</span></span>  
 <span data-ttu-id="5ee9a-117">È possibile usare la clausola EXECUTE AS nell'intestazione della definizione di una stored procedure, un trigger o una funzione definita dall'utente, ad eccezione delle funzioni inline valutate a livello di tabella.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-117">You can use the EXECUTE AS clause in the definition header of a stored procedure, trigger, or user-defined function (except for inline table-valued functions).</span></span> <span data-ttu-id="5ee9a-118">La stored procedure viene quindi eseguita nel contesto del nome utente o della parola chiave specificata nella clausola EXECUTE AS.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-118">This causes the procedure to execute in the context of the user name or keyword specified in the EXECUTE AS clause.</span></span> <span data-ttu-id="5ee9a-119">È possibile creare un utente proxy nel database di cui non è stato eseguito il mapping a un account di accesso, concedendo solo le autorizzazioni necessarie sugli oggetti cui la stored procedure ha accesso.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-119">You can create a proxy user in the database that is not mapped to a login, granting it only the necessary permissions on the objects accessed by the procedure.</span></span> <span data-ttu-id="5ee9a-120">Solo l'utente proxy specificato nella clausola EXECUTE AS deve disporre delle autorizzazioni su tutti gli oggetti cui il modulo ha accesso.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-120">Only the proxy user specified in the EXECUTE AS clause must have permissions on all objects accessed by the module.</span></span>  
  
> [!NOTE]
> <span data-ttu-id="5ee9a-121">Per alcune azioni, ad esempio TRUNCATE TABLE, non sono disponibili autorizzazioni da concedere.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-121">Some actions, such as TRUNCATE TABLE, do not have grantable permissions.</span></span> <span data-ttu-id="5ee9a-122">Incorporando l'istruzione all'interno di una stored procedure e specificando un utente proxy che dispone delle autorizzazioni ALTER TABLE, è possibile estendere le autorizzazioni necessarie per troncare la tabella ai chiamanti che dispongono solo delle autorizzazioni EXECUTE per la stored procedure.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-122">By incorporating the statement within a procedure and specifying a proxy user who has ALTER TABLE permissions, you can extend the permissions to truncate the table to callers who have only EXECUTE permissions on the procedure.</span></span>  
  
 <span data-ttu-id="5ee9a-123">Il contesto specificato nella clausola EXECUTE AS è valido per la durata della stored procedura, incluse stored procedure e trigger annidati.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-123">The context specified in the EXECUTE AS clause is valid for the duration of the procedure, including nested procedures and triggers.</span></span> <span data-ttu-id="5ee9a-124">Il contesto ritorna al chiamante al termine dell'esecuzione o quando viene eseguita l'istruzione REVERT.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-124">Context reverts to the caller when execution is complete or the REVERT statement is issued.</span></span>  
  
 <span data-ttu-id="5ee9a-125">L'uso di una clausola EXECUTE AS in una stored procedure implica tre passaggi:</span><span class="sxs-lookup"><span data-stu-id="5ee9a-125">There are three steps involved in using the EXECUTE AS clause in a procedure.</span></span>  
  
1. <span data-ttu-id="5ee9a-126">Creazione di un utente proxy nel database non mappato a un account di accesso.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-126">Create a proxy user in the database that is not mapped to a login.</span></span> <span data-ttu-id="5ee9a-127">Questo passaggio non è necessario ma risulta utile durante la gestione delle autorizzazioni.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-127">This is not required, but it helps when managing permissions.</span></span>

1. <span data-ttu-id="5ee9a-128">Concessione delle autorizzazioni necessarie all'utente proxy.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-128">Grant the proxy user the necessary permissions.</span></span>

> [!NOTE]
> <span data-ttu-id="5ee9a-130">È possibile che le applicazioni che richiedono il controllo vengano interrotte perché il contesto di sicurezza originale del chiamante non viene mantenuto.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-130">Applications that require auditing can break because the original security context of the caller is not retained.</span></span> <span data-ttu-id="5ee9a-131">Le funzioni predefinite che restituiscono l'identità dell'utente corrente, ad esempio SESSION_USER, USER, o USER_NAME, restituiscono l'utente associato alla clausola EXECUTE AS, non il chiamante originale.</span><span class="sxs-lookup"><span data-stu-id="5ee9a-131">Built-in functions that return the identity of the current user, such as SESSION_USER, USER, or USER_NAME, return the user associated with the EXECUTE AS clause, not the original caller.</span></span>
