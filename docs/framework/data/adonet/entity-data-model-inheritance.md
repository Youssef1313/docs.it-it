# <a name="entity-data-model-inheritance"></a><span data-ttu-id="e9ad6-102">Entity Data Model: ereditarietà</span><span class="sxs-lookup"><span data-stu-id="e9ad6-102">Entity Data Model: Inheritance</span></span>
<span data-ttu-id="e9ad6-103">Il Entity Data Model (EDM) supporta l'ereditarietà per i [tipi di entità](entity-type.md).</span><span class="sxs-lookup"><span data-stu-id="e9ad6-103">The Entity Data Model (EDM) supports inheritance for [entity types](entity-type.md).</span></span> <span data-ttu-id="e9ad6-104">L'ereditarietà in EDM è analoga all'ereditarietà per le classi nei linguaggi di programmazione orientati a oggetti.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-104">Inheritance in the EDM is similar to inheritance for classes in object-oriented programming languages.</span></span> <span data-ttu-id="e9ad6-105">Analogamente a quanto avviene con le classi nei linguaggi orientati a oggetti, in un modello concettuale è possibile definire un tipo di entità (un *tipo derivato*) che eredita da un altro tipo di entità (il *tipo di base*).</span><span class="sxs-lookup"><span data-stu-id="e9ad6-105">Like with classes in object-oriented languages, in a conceptual model you can define an entity type (a *derived type*) that inherits from another entity type (the *base type*).</span></span> <span data-ttu-id="e9ad6-106">Tuttavia, a differenza delle classi nella programmazione orientata a oggetti, in un modello concettuale il tipo derivato eredita sempre tutte le [Proprietà](property.md) e le [proprietà di navigazione](navigation-property.md) del tipo di base.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-106">However, unlike classes in object-oriented programming, in a conceptual model the derived type always inherits all the [properties](property.md) and [navigation properties](navigation-property.md) of the base type.</span></span> <span data-ttu-id="e9ad6-107">Non è possibile eseguire l'override delle proprietà ereditate in un tipo derivato.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-107">You cannot override inherited properties in a derived type.</span></span>  
  
 <span data-ttu-id="e9ad6-108">In un modello concettuale è possibile compilare gerarchie di ereditarietà nelle quali un tipo derivato eredita da un altro tipo derivato.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-108">In a conceptual model you can build inheritance hierarchies in which a derived type inherits from another derived type.</span></span> <span data-ttu-id="e9ad6-109">Il tipo all'inizio della gerarchia (il tipo nella gerarchia che non è un tipo derivato) viene definito *tipo radice*.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-109">The type at the top of the hierarchy (the one type in the hierarchy that is not a derived type) is called the *root type*.</span></span> <span data-ttu-id="e9ad6-110">In una gerarchia di ereditarietà, la [chiave di entità](entity-key.md) deve essere definita nel tipo radice.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-110">In an inheritance hierarchy, the [entity key](entity-key.md) must be defined on the root type.</span></span>  
  
 <span data-ttu-id="e9ad6-111">Non è possibile compilare gerarchie di ereditarietà nelle quali un tipo derivato eredita da più di un tipo.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-111">You cannot build inheritance hierarchies in which a derived type inherits from more than one type.</span></span> <span data-ttu-id="e9ad6-112">In un modello concettuale con un tipo di entità `Book`, ad esempio, è possibile definire i tipi derivati `FictionBook` e `NonFictionBook` che ereditano ognuno da `Book`.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-112">For example, in a conceptual model with a `Book` entity type, you could define derived types `FictionBook` and `NonFictionBook` that each inherit from `Book`.</span></span> <span data-ttu-id="e9ad6-113">Non è quindi possibile, tuttavia, definire un tipo che eredita sia dal tipo `FictionBook` che dal tipo `NonFictionBook`.</span><span class="sxs-lookup"><span data-stu-id="e9ad6-113">However, you could not then define a type that inherits from both the `FictionBook` and `NonFictionBook` types.</span></span>
