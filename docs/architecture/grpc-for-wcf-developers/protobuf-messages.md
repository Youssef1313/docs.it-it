## <a name="declaring-a-message"></a><span data-ttu-id="4500d-106">Dichiarazione di un messaggio</span><span class="sxs-lookup"><span data-stu-id="4500d-106">Declaring a message</span></span>

<span data-ttu-id="4500d-107">In WCF è possibile definire una classe `Stock` per un'applicazione di trading sul mercato azionario come nell'esempio seguente:</span><span class="sxs-lookup"><span data-stu-id="4500d-107">In WCF, a `Stock` class for a stock market trading application might be defined like the following example:</span></span>

```csharp
namespace TraderSys
{
    [DataContract]
    public class Stock
    {
        [DataMember]
        public int Id { get; set;}
        [DataMember]
        public string Symbol { get; set;}
        [DataMember]
        public string DisplayName { get; set;}
        [DataMember]
        public int MarketId { get; set; }
    }
}
```

<span data-ttu-id="4500d-108">Per implementare la classe equivalente in protobuf, deve essere dichiarata nel file di `.proto`.</span><span class="sxs-lookup"><span data-stu-id="4500d-108">To implement the equivalent class in Protobuf, it must be declared in the `.proto` file.</span></span> <span data-ttu-id="4500d-109">Il compilatore `protoc` genererà la classe .NET come parte del processo di compilazione.</span><span class="sxs-lookup"><span data-stu-id="4500d-109">The `protoc` compiler will then generate the .NET class as part of the build process.</span></span>

```protobuf
syntax "proto3";

option csharp_namespace = "TraderSys";

message Stock {

    int32 id = 1;
    string symbol = 2;
    string display_name = 3;
    int32 market_id = 4;

}  
```

<span data-ttu-id="4500d-110">Nella prima riga viene dichiarata la versione della sintassi utilizzata.</span><span class="sxs-lookup"><span data-stu-id="4500d-110">The first line declares the syntax version being used.</span></span> <span data-ttu-id="4500d-111">La versione 3 della lingua è stata rilasciata nel 2016 ed è la versione consigliata per i servizi gRPC.</span><span class="sxs-lookup"><span data-stu-id="4500d-111">Version 3 of the language was released in 2016 and is the recommended version for gRPC services.</span></span>

<span data-ttu-id="4500d-112">La riga di `option csharp_namespace` specifica lo spazio dei nomi da utilizzare per C# i tipi generati.</span><span class="sxs-lookup"><span data-stu-id="4500d-112">The `option csharp_namespace` line specifies the namespace to be used for the generated C# types.</span></span> <span data-ttu-id="4500d-113">Questa opzione verrà ignorata quando il file di `.proto` viene compilato per altre lingue.</span><span class="sxs-lookup"><span data-stu-id="4500d-113">This option will be ignored when the `.proto` file is compiled for other languages.</span></span> <span data-ttu-id="4500d-114">È comune che i file protobuf contengano opzioni specifiche della lingua per diverse lingue.</span><span class="sxs-lookup"><span data-stu-id="4500d-114">It is common for Protobuf files to contain language-specific options for several languages.</span></span>

<span data-ttu-id="4500d-115">Nella definizione del messaggio `Stock` sono specificati quattro campi, ognuno con un tipo, un nome e un numero di campo.</span><span class="sxs-lookup"><span data-stu-id="4500d-115">The `Stock` message definition specifies four fields, each with a type, a name, and a field number.</span></span>

## <a name="field-numbers"></a><span data-ttu-id="4500d-116">Numeri di campo</span><span class="sxs-lookup"><span data-stu-id="4500d-116">Field numbers</span></span>

<span data-ttu-id="4500d-117">I numeri di campo sono una parte importante di protobuf.</span><span class="sxs-lookup"><span data-stu-id="4500d-117">Field numbers are an important part of Protobuf.</span></span> <span data-ttu-id="4500d-118">Vengono usati per identificare i campi nei dati con codifica binaria, che significa che non possono essere modificati dalla versione alla versione del servizio.</span><span class="sxs-lookup"><span data-stu-id="4500d-118">They're used to identify fields in the binary encoded data, which means they can't change from version to version of your service.</span></span> <span data-ttu-id="4500d-119">Il vantaggio è che è possibile la compatibilità con le versioni precedenti e precedenti.</span><span class="sxs-lookup"><span data-stu-id="4500d-119">The advantage is that backward and forward compatibility is possible.</span></span> <span data-ttu-id="4500d-120">I client e i servizi ignoreranno semplicemente i numeri di campo che non conoscono, purché venga gestita la possibilità di valori mancanti.</span><span class="sxs-lookup"><span data-stu-id="4500d-120">Clients and services will simply ignore field numbers they don't know about, as long as the possibility of missing values is handled.</span></span>
